# 任务记录：安全中心失败事件统计修正

## 任务状态

已完成

## 任务背景

安全中心展示“失败事件”数量，用于帮助用户快速发现异常操作。后端操作日志动作命名使用点号形式，例如 `server.test_ssh.failed`，但前端当前只匹配 `_failed` 后缀，导致失败事件统计不准确。

## 本次任务目标

修正安全中心失败事件计数规则，兼容当前后端操作日志动作命名。

## 本次调整范围

### 前端

- 失败事件统计同时匹配 `.failed` 和 `_failed` 后缀。

## 不包含范围

- 不调整后端操作日志动作命名。
- 不增加复杂筛选。

## 验收标准

- `server.test_ssh.failed` 能被统计为失败事件。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 当前仅按动作后缀统计失败事件，后续如新增失败动作需要保持命名约定。

## 完成记录

- 安全中心失败事件统计已兼容 `.failed` 和 `_failed` 后缀。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
