# 任务记录：概览工作台接入真实数据

## 任务状态

已完成

## 任务背景

根据 `PRD.md`，系统第一期需要让用户集中管理服务器、协议节点、订阅和安装/卸载任务。当前概览页仍展示脚手架占位数据，不利于用户登录后快速了解系统状态，也不满足已实现功能的入口聚合需求。

## 本次任务目标

将前端概览页从静态占位改为真实工作台，接入服务器、节点、订阅、任务接口，展示关键数量、最近任务和最近节点，并提供进入核心流程的快捷入口。

## 本次调整范围

### 前端

- 概览页通过 React Query 拉取真实服务器、节点、订阅和任务数据。
- 统计卡展示真实数量和运行中任务数。
- 展示最近任务状态。
- 展示最近协议节点状态。
- 快捷按钮跳转到服务器添加、外部节点导入/系统安装、订阅管理等页面。
- 移除“脚手架页面”占位文案。

## 不包含范围

- 不新增后端统计接口。
- 不实现复杂图表。
- 不调整服务器、节点、订阅页面现有业务表单。

## 验收标准

- 概览页数量来自真实接口。
- 概览页不再出现脚手架占位描述。
- 用户可以从概览页快速进入核心功能页面。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 第一版直接复用列表接口统计数量，数据量很大时后续可增加后端聚合统计接口。

## 完成记录

- 概览页已接入服务器、协议节点、订阅和任务列表接口。
- 统计卡已改为真实数量，包含可用节点和运行中任务。
- 已新增最近任务、最近节点展示。
- 已新增服务器、节点、订阅快捷入口。
- 已移除脚手架占位文案。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
