# 任务记录：外部协议节点基础管理

## 任务状态

已完成

## 任务背景

根据 `PRD.md`，用户除了通过系统 SSH 自动安装协议节点外，也可以直接添加外部服务器已经安装好的节点，并保存节点信息到数据库中，用于后续统一生成订阅。当前已完成用户认证、服务器管理和 NAT 端口映射管理，下一步应先实现不依赖远程脚本执行的协议节点基础管理能力。

## 本次任务目标

实现普通用户的协议节点列表、外部节点手动添加、分享链接导入、编辑和删除，为后续订阅管理提供节点数据基础。

## 本次调整范围

### 后端

- 实现节点列表接口：`GET /api/v1/nodes`
- 实现外部节点导入接口：`POST /api/v1/nodes/import`
- 实现节点编辑接口：`PUT /api/v1/nodes/:id`
- 实现节点删除接口：`DELETE /api/v1/nodes/:id`
- 支持两种导入方式：
  - 手动填写节点参数
  - 粘贴分享链接导入
- 外部节点不要求绑定系统服务器。
- 节点敏感配置使用 `ENCRYPTION_KEY` 加密保存。
- 接口响应不返回敏感配置明文。
- 所有节点接口按当前登录用户隔离数据。
- 增加基础测试覆盖外部节点 CRUD 和跨用户隔离。

### 前端

- 实现协议节点页面。
- 展示当前用户自己的节点列表。
- 支持手动添加外部节点。
- 支持粘贴分享链接导入外部节点。
- 支持编辑节点基础信息。
- 支持删除节点。
- 展示节点协议、地址、端口、状态和来源。

## 不包含范围

- 不实现通过 SSH 自动安装协议。
- 不实现卸载协议脚本执行。
- 不实现任务队列。
- 不实现订阅生成。
- 不实现所有协议分享链接的完整参数解析；第一期先保存原始链接并解析基础协议、地址、端口、名称。

## 验收标准

- 用户可以查看自己的协议节点列表。
- 用户可以手动添加外部节点。
- 用户可以通过分享链接导入外部节点。
- 用户可以编辑自己的外部节点。
- 用户可以删除自己的外部节点。
- 用户不能查看或操作其他用户的节点。
- 节点敏感配置不会明文返回给前端。
- 前端协议节点页面可以完成列表、添加、导入、编辑、删除操作。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 分享链接格式很多，本任务只做基础解析和原始链接保存，完整客户端格式转换在订阅生成任务中继续完善。
- 系统安装节点状态流转会在后续“协议安装任务”中实现。
