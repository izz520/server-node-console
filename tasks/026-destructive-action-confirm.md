# 任务记录：破坏性操作确认

## 任务状态

已完成

## 任务背景

服务器删除、节点删除、节点卸载、订阅删除、订阅 token 重置、NAT 映射删除都属于不可逆或有明显影响的操作。当前前端点击按钮后直接执行，容易误操作。

## 本次任务目标

为破坏性操作增加确认提示，降低误删、误卸载和误重置 token 的风险。

## 本次调整范围

### 前端

- 删除服务器前确认。
- 删除 NAT 映射前确认。
- 删除节点前确认。
- 卸载系统节点前确认。
- 删除订阅前确认。
- 重置订阅 token 前确认。

## 不包含范围

- 不实现自定义 Modal。
- 不改变后端接口。
- 不增加批量操作。

## 验收标准

- 用户取消确认时不会发起请求。
- 用户确认后保持原有操作流程。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 第一版使用浏览器原生确认框，后续可替换为统一设计的确认弹窗。
