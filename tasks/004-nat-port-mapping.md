# 任务记录：NAT 端口映射管理

## 任务状态

已完成

## 任务背景

根据 `PRD.md`，NAT 端口映射是系统内记录功能，不真实修改服务器网络配置。它用于记录节点实际监听端口和公网对外访问端口的关系，后续订阅生成时需要优先使用对外访问端口。

服务器管理能力已完成，因此下一个任务应实现服务器下的 NAT 端口映射管理。

## 本次任务目标

实现普通用户对自己服务器下 NAT 端口映射记录的增删改查，并在前端服务器管理页面中提供映射维护入口。

## 本次调整范围

### 后端

- 实现 NAT 映射列表接口：`GET /api/v1/servers/:id/nat-mappings`
- 实现 NAT 映射创建接口：`POST /api/v1/servers/:id/nat-mappings`
- 实现 NAT 映射编辑接口：`PUT /api/v1/nat-mappings/:id`
- 实现 NAT 映射删除接口：`DELETE /api/v1/nat-mappings/:id`
- 所有接口校验服务器和映射记录必须属于当前登录用户。
- 字段包括：映射名称、协议类型、服务实际监听端口、对外访问端口、备注。
- 增加基础测试覆盖用户数据隔离和 CRUD。

### 前端

- 在服务器管理页面增加 NAT 映射管理区域。
- 用户可选择某台服务器查看其 NAT 映射。
- 支持添加、编辑、删除 NAT 映射。
- 显示服务实际监听端口和对外访问端口。
- 明确提示：NAT 映射只是系统记录，不会真实修改服务器配置。

## 不包含范围

- 不真实修改服务器网络配置。
- 不实现协议节点安装。
- 不实现订阅生成逻辑。
- 不实现 NAT 映射与具体节点的自动绑定。

## 验收标准

- 用户可以查看自己服务器下的 NAT 映射列表。
- 用户可以为自己的服务器创建 NAT 映射。
- 用户可以编辑自己的 NAT 映射。
- 用户可以删除自己的 NAT 映射。
- 用户不能查看或操作其他用户服务器下的 NAT 映射。
- NAT 映射字段包含服务实际监听端口和对外访问端口。
- 前端可以在服务器管理页面完成 NAT 映射维护。
- 页面明确提示 NAT 映射不会真实修改服务器配置。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 协议类型建议第一期作为可选文本字段，默认支持 TCP/UDP 文本输入或选择。
- 是否限制同一服务器下重复的监听端口/对外端口组合，建议第一期先不强制唯一。
