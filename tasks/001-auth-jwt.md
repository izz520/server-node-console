# 任务记录：用户注册登录与 JWT 鉴权

## 任务状态

已完成

## 任务背景

根据 `PRD.md`，系统第一期需要支持多用户 SaaS 能力。普通用户需要注册、登录，并且后续服务器、节点、订阅、任务日志都必须按用户维度隔离。因此下一个开发任务应优先实现认证基础能力。

## 本次任务目标

实现后端用户注册、登录、当前用户信息查询和 JWT 鉴权闭环，并让前端登录页从占位登录切换为真实接口调用。

## 本次调整范围

### 后端

- 实现用户注册接口：`POST /api/v1/auth/register`
- 实现用户登录接口：`POST /api/v1/auth/login`
- 实现当前用户接口：`GET /api/v1/me`
- 增加密码哈希与校验逻辑
- 增加 JWT 生成逻辑
- 登录 JWT 过期时间设置为 7 天
- 增加续期接口，用于前端自动续期 token
- 保持普通用户默认角色为 `user`
- 为注册、登录、当前用户接口增加基础测试

### 前端

- 登录页调用真实登录接口
- 登录成功后保存 JWT 和用户信息
- 页面刷新后通过 `GET /api/v1/me` 恢复用户状态
- 前端在 token 临近过期时自动调用续期接口刷新 token
- 前端收到 401 时清理登录状态并跳转登录页
- 增加注册页面或在登录页提供注册入口
- 未登录用户继续跳转到登录页

## 不包含范围

- 管理员用户创建方式
- 邮箱验证码
- 登录失败次数限制和账号锁定
- 用户禁用功能
- 服务器、节点、订阅业务接口实现

## 验收标准

- 用户可以通过用户名、邮箱和密码注册账号。
- 重复用户名或重复邮箱注册失败。
- 数据库不保存明文密码。
- 用户可以使用用户名或邮箱登录。
- 登录成功后返回 JWT 和用户信息。
- JWT 过期时间为 7 天。
- 前端登录成功后进入控制台。
- 未登录访问控制台会跳转到登录页。
- 页面刷新后可通过 JWT 恢复当前用户。
- 前端在 token 临近过期时自动续期。
- 续期成功后前端使用新 token 继续访问接口。
- 无效 JWT 无法访问受保护接口。

## 风险与待确认

- 管理员账号第一期是否通过数据库手动创建，暂不做后台创建。
- 自动续期策略建议：token 剩余有效期小于 24 小时时触发续期。
- 续期接口建议复用当前 JWT 鉴权，不额外引入 refresh token；如需 refresh token 机制，请确认。
