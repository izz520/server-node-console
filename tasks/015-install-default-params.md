# 任务记录：协议安装默认参数生成

## 任务状态

已完成

## 任务背景

`PRD.md` 要求协议参数能自动生成的尽量自动生成，例如 UUID、密码、密钥等；用户不填写时系统使用自动生成值或协议默认值。当前安装接口对端口和 UUID 的处理偏基础，且任务日志可能记录完整安装命令，不利于敏感参数保护。

## 本次任务目标

在后端协议安装入口补齐默认参数生成与敏感参数保护：用户留空端口时生成可用端口，协议需要 UUID 且用户留空时自动生成 UUID，并将生成参数加密保存、任务日志脱敏。

## 本次调整范围

### 后端

- 安装请求字段统一 trim。
- 端口为空时生成随机高位端口。
- VLESS/VMess/Reality/Argo/AnyTLS 等协议留空 UUID 时自动生成 UUID v4。
- 将生成或填写的敏感安装参数加密保存到节点的 `EncryptedProtocolJSON`。
- 任务日志中的安装命令脱敏 UUID、Argo token 等敏感参数。
- 增加测试覆盖默认端口、UUID 自动生成、敏感参数加密标记。

## 不包含范围

- 不完整生成所有协议的专属密钥。
- 不在前端回显生成后的敏感参数明文。
- 不实现安装失败后的参数重试。

## 验收标准

- 用户安装支持协议时可不填写 UUID。
- 用户安装时可不填写端口，后端自动生成合法端口。
- 自动生成或填写的敏感参数不会明文返回前端。
- 任务日志不会记录敏感参数明文。
- `go test ./...` 通过。

## 风险与待确认

- 第一版先覆盖 UUID 和常见 token，后续可按协议逐步补齐密码、公私钥等专属参数。

## 完成记录

- 安装请求字段已统一 trim。

- VLESS/VMess/Reality/Argo/AnyTLS 等协议未填写 UUID 时，后端自动生成 UUID v4。
- 自动生成或填写的 UUID、Argo token 已加密保存到节点敏感参数字段。
- 安装任务日志中的 UUID、Argo token 已脱敏。
- 已增加测试覆盖默认端口和敏感参数加密标记。
