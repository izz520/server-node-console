# 任务记录：服务器删除保护补充 NAT 映射

## 任务状态

已完成

## 任务背景

服务器删除保护已经覆盖节点和订阅引用，但 NAT 端口映射也是服务器下属数据。如果服务器存在 NAT 映射时仍允许删除，会留下失去服务器归属的 NAT 记录。

## 本次任务目标

删除服务器前检查 NAT 映射，存在 NAT 映射时拒绝删除并提示用户先删除映射。

## 本次调整范围

### 后端

- `DeleteServer` 保护逻辑增加 NAT 映射检查。
- 增加测试覆盖存在 NAT 映射时服务器删除失败。

### 前端

- 复用现有错误提示展示后端返回原因。

## 不包含范围

- 不自动级联删除 NAT 映射。
- 不新增强制删除。

## 验收标准

- 存在 NAT 映射的服务器删除失败。
- 删除 NAT 映射后服务器可继续按原规则删除。
- `go test ./...` 通过。

## 风险与待确认

- 第一版继续采用保护策略，避免隐式删除用户配置。

## 完成记录

- 服务器删除保护已增加 NAT 映射检查。
- 已增加 NAT 映射存在时服务器删除失败的后端测试。
- 前端常见错误映射已补充 NAT 映射保护提示。

## 验证结果
