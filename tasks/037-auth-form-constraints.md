# 任务记录：登录注册表单基础约束

## 任务状态

已完成

## 任务背景

后端注册要求用户名、邮箱、密码必填，密码至少 8 位；登录要求账号和密码必填。当前前端登录/注册表单缺少部分 HTML 表单约束，用户可能提交明显无效的数据后才看到后端错误。

## 本次任务目标

补齐登录和注册表单的基础浏览器校验，减少无效请求。

## 本次调整范围

### 前端

- 登录账号和密码输入框增加 `required`。
- 注册用户名、邮箱、密码输入框增加 `required`。
- 注册密码输入框增加 `minLength={8}`。

## 不包含范围

- 不实现复杂密码强度校验。
- 不新增字段级错误组件。

## 验收标准

- 空账号/密码无法直接提交登录。
- 注册密码少于 8 位时浏览器提示。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 浏览器原生校验文案由浏览器决定；后续可统一做自定义表单校验。

## 完成记录

- 登录账号和密码输入框已增加 `required`。
- 注册用户名、邮箱、密码输入框已增加 `required`。
- 注册密码输入框已增加 `minLength={8}`。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
