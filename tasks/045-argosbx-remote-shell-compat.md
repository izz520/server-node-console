# 任务记录：修复 argosbx 远端命令 shell 兼容性

## 任务状态

已完成

## 任务背景

真实 AnyTLS 安装端到端验证中，后端已成功创建安装任务并通过 SSH 执行命令，但远端返回 `status 127`。当前生成命令直接使用 `bash <(curl ...)`，当 SSH 会话先经过远端默认 shell 解析时，进程替换语法可能无法被支持，导致脚本未能进入 argosbx 安装逻辑。

## 本次任务目标

让 argosbx 安装/卸载命令先通过 POSIX `sh` 检查远端依赖，缺少 `bash` 或 `curl` 时自动尝试用常见包管理器安装，然后显式通过 `bash -lc` 执行 argosbx，并补充命令生成测试，保证包含空格、单引号等参数时仍能正确传给远端 Bash。

## 本次调整范围

### 后端

- 调整 `backend/internal/argosbx/scripts.go` 的安装命令生成逻辑。
- 调整卸载命令生成逻辑。
- 增加远端 `bash/curl` 依赖引导逻辑。
- 增加 argosbx 命令生成单元测试。

### 验证

- 运行后端测试。
- 重新跑 AnyTLS 真实安装/卸载端到端验证。

## 不包含范围

- 不改变 argosbx 脚本源。
- 不在仓库中保存 SSH 密码。
- 不修改协议参数表之外的业务流程。

## 验收标准

- 生成的安装命令以 `sh -lc` 执行依赖引导，并通过 `bash -lc` 包裹远端脚本。
- 远端缺少 `bash` 或 `curl` 时会自动尝试安装。
- 变量值可安全包含空格和单引号。
- `go test ./...` 通过。
- AnyTLS 真实安装验证不再因为默认 shell 不支持进程替换而失败。

## 执行结果

- 已将安装/卸载命令改为 `sh -lc` 依赖引导 + `bash -lc` 执行。
- 真实服务器原始失败原因为缺少 `bash`，修复后已能进入 argosbx 脚本执行阶段。
- 后端测试通过。
