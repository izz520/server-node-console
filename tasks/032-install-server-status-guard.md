# 任务记录：安装协议前校验服务器状态

## 任务状态

已完成

## 任务背景

`PRD.md` 中服务器状态定义为：正常表示 SSH 可连接、可执行安装/卸载；连接失败表示最近一次 SSH 测试失败；禁用表示暂不允许用于安装/卸载。当前安装接口只校验服务器归属和协议支持，没有拦截非正常状态的服务器。

## 本次任务目标

在发起协议安装前校验服务器状态，只有状态为 `normal` 的服务器允许创建安装任务。

## 本次调整范围

### 后端

- `InstallNode` 在创建节点和任务前检查服务器状态。
- 连接失败或禁用服务器返回 `400 Bad Request`。
- 增加测试覆盖非正常服务器不能发起安装。

### 前端

- 复用现有错误提示展示后端返回原因。
- 补充常见错误中文映射。

## 不包含范围

- 不自动触发 SSH 测试。
- 不改变服务器状态更新逻辑。
- 不实现服务器禁用开关。

## 验收标准

- 正常服务器可以发起安装。
- 连接失败服务器不能发起安装。
- 禁用服务器不能发起安装。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 用户需要先在服务器页面测试连通性恢复正常后再安装协议。
