# 任务记录：服务器删除保护

## 任务状态

已完成

## 任务背景

`PRD.md` 在服务器删除规则中建议第一期采用保护策略：如果服务器下存在已安装节点或订阅引用，不允许直接删除。当前服务器删除接口缺少关联数据检查，可能导致节点失去服务器归属，影响后续卸载和订阅管理。

## 本次任务目标

为服务器删除接口增加关联保护：当服务器下仍存在协议节点，或这些节点仍被订阅引用时，拒绝删除并返回明确错误提示。

## 本次调整范围

### 后端

- 删除服务器前检查当前用户该服务器下是否存在节点。
- 删除服务器前检查该服务器节点是否被订阅引用。
- 有关联数据时返回 `409 Conflict` 和可读错误。
- 增加后端测试覆盖可删除、有关联节点不可删除、订阅引用不可删除。

### 前端

- 复用现有错误提示机制展示后端返回的删除失败原因。

## 不包含范围

- 不做级联删除。
- 不自动从订阅中移除节点。
- 不自动卸载节点。

## 验收标准

- 无关联数据的服务器可以删除。
- 有节点的服务器删除失败。
- 有订阅引用节点的服务器删除失败。
- `go test ./...` 通过。

## 风险与待确认

- 第一版按保护策略处理，不引入强制删除能力。
