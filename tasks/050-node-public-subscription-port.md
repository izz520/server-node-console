# 任务记录：节点级公网订阅端口

## 任务状态

已完成

## 任务背景

NAT 服务器上的协议节点通常有两个端口：服务端真实监听端口和服务商映射后的公网端口。例如节点实际安装监听 `43888`，但服务商提供的公网端口是 `48888`。订阅链接应使用公网端口，否则客户端无法连通。

项目已有 `ProtocolNode.PublicPort` 字段，并且订阅渲染已优先使用 `PublicPort`。但系统安装节点没有直接填写公网订阅端口的入口，安装后的系统节点也不能编辑该订阅端口，用户需要绕到独立 NAT 映射功能，和实际订阅场景不够贴合。

## 本次任务目标

把公网端口映射能力收敛到协议节点自身：用户在安装或编辑节点时直接填写“公网订阅端口”，订阅链接自动优先使用该端口。

## 本次调整范围

### 后端

- 系统安装节点请求支持 `publicPort`。
- 创建系统安装节点时保存 `PublicPort`。
- 校验 `publicPort` 必须在 `1-65535` 范围内。
- 系统安装节点允许编辑订阅展示信息：
  - 节点名称
  - 备注
  - 公网订阅端口
- 系统安装节点禁止编辑核心配置：
  - 协议
  - 地址
  - 安装/监听端口
- 保持订阅渲染优先使用 `node.PublicPort` 的规则。

### 前端

- 系统安装表单增加“公网订阅端口”字段。
- 节点列表同时展示真实监听端口和订阅端口。
- 系统安装节点也显示编辑入口。
- 编辑系统安装节点时锁定协议、地址、端口等核心字段，只开放订阅展示相关字段。
- 手动节点字段文案统一为“公网订阅端口”。

### 测试

- 增加系统安装节点保存公网订阅端口的测试。
- 增加系统安装节点允许修改公网订阅端口的测试。
- 增加系统安装节点拒绝修改核心配置的测试。

## 不包含范围

- 不移除已有独立 NAT 映射接口和页面。
- 不自动把已有 NAT 映射迁移到节点 `publicPort`。
- 不自动探测服务商公网端口。
- 不修改服务器上的真实监听端口。

## 验收标准

- 安装节点时填写 `port=43888`、`publicPort=48888` 后，节点记录保存公网订阅端口。
- 订阅链接使用 `48888`，而不是真实监听端口 `43888`。
- 已安装系统节点可在节点页编辑公网订阅端口。
- 系统安装节点核心配置被后端保护，误改端口或协议会返回错误。
- 前端构建通过。
- 后端 handler 测试通过。
