# 任务记录：系统安装节点删除保护

## 任务状态

已完成

## 任务背景

`PRD.md` 要求卸载协议时系统通过 SSH 自动执行卸载脚本，并在成功后删除或标记记录。当前 `DeleteNode` 允许直接删除任意节点，系统安装成功的节点可能绕过卸载脚本，导致服务器上实际服务仍存在。

## 本次任务目标

保护系统安装节点：未卸载的系统安装节点不能直接删除，用户必须先发起卸载任务。外部导入节点和已卸载系统节点仍可删除记录。

## 本次调整范围

### 后端

- `DeleteNode` 检查节点安装方式和状态。
- 系统安装且状态不是 `uninstalled` 的节点返回 `400 Bad Request`。
- 增加测试覆盖系统安装成功节点不能直接删除。

### 前端

- 节点列表中，未卸载系统节点隐藏删除按钮，只保留卸载入口。
- 补充中文错误映射。

## 不包含范围

- 不实现强制删除服务器记录。
- 不改变外部节点删除逻辑。
- 不改变卸载任务执行逻辑。

## 验收标准

- 系统安装成功节点不能直接删除。
- 外部导入节点可以删除。
- 已卸载系统节点可以删除记录。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 如果服务器永久不可达，后续可能需要“仅删除记录”的特殊运维能力；第一期先按 PRD 保护。

## 完成记录

- `DeleteNode` 已禁止直接删除未卸载的系统安装节点。
- 外部导入节点和已卸载系统节点仍允许删除记录。
- 节点列表已对未卸载系统节点隐藏删除按钮。
- 前端错误中文映射已补充对应提示。
- 已增加后端测试覆盖系统安装成功节点不能直接删除。
