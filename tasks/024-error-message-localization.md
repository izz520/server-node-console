# 任务记录：常见接口错误中文化

## 任务状态

已完成

## 任务背景

前端已经可以展示后端返回的错误信息，但后端当前错误多为英文短语，例如 `username or email already exists`、`invalid account or password`。系统主要面向中文界面用户，需要把常见接口错误映射为中文提示。

## 本次任务目标

在前端 API 错误解析 helper 中增加常见错误中文映射，提升注册、登录、服务器、节点、订阅等页面的错误可读性。

## 本次调整范围

### 前端

- 为常见后端错误增加中文映射。
- 保留未知错误原文或兜底文案。

## 不包含范围

- 不调整后端错误结构。
- 不引入 i18n 框架。
- 不覆盖所有可能的第三方 SSH 错误原文。

## 验收标准

- 用户名/邮箱重复显示中文提示。
- 账号或密码错误显示中文提示。
- 常见资源不存在、权限、订阅禁用等错误显示中文提示。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 第一版按当前后端错误短语映射；后续可改为错误码驱动。

## 完成记录

- API 错误解析 helper 已增加常见错误中文映射。
- 注册重复、登录失败、资源不存在、订阅禁用、服务器删除保护等错误会显示中文提示。
- 未知错误仍保留原文或兜底文案。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
