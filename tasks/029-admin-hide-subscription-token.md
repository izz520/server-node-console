# 任务记录：管理员订阅列表隐藏 token

## 任务状态

已完成

## 任务背景

`PRD.md` 要求管理员第一期只读查看用户数据，但不允许查看 SSH 密码、私钥、节点密钥等敏感明文。订阅 token 是匿名访问订阅链接的敏感凭据，管理员订阅列表当前会复用普通用户响应并解密返回 token，存在越权可见风险。

## 本次任务目标

管理员订阅列表不返回订阅 token 和订阅链接，只展示订阅名称、格式、启用状态、节点数、用户归属等非敏感信息。

## 本次调整范围

### 后端

- `AdminListSubscriptions` 调用订阅响应构建时不包含 token。
- 增加测试覆盖管理员订阅列表不返回 token。

## 不包含范围

- 不改变普通用户查看自己订阅链接的能力。
- 不改变公共订阅 `/sub/{token}` 访问逻辑。

## 验收标准

- 普通用户订阅列表仍能看到自己的订阅链接。
- 管理员订阅列表不返回 token 和 subscriptionUrl。
- `go test ./...` 通过。

## 风险与待确认

- 管理员若需要排查订阅内容，可通过用户提供链接或后续增加脱敏诊断工具。

## 完成记录

- 管理员订阅列表已改为不解密、不返回 token 和 subscriptionUrl。
- 普通用户订阅列表能力保持不变。
- 已增加测试覆盖管理员订阅列表不暴露 token。
