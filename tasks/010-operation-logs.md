# 任务记录：操作日志记录与查看

## 任务状态

已完成

## 任务背景

根据 `PRD.md` 的安全要求，系统建议记录关键操作日志，包括用户登录、服务器增删改、SSH 测试、节点安装/卸载、外部节点导入、订阅增删改和 token 重置。当前业务主流程已基本实现，需要补齐操作日志能力，便于后续审计和问题排查。

## 本次任务目标

实现后端操作日志记录工具，在关键接口中写入操作日志，并提供当前用户和管理员只读查看操作日志的能力。

## 本次调整范围

### 后端

- 增加操作日志写入 helper。
- 在以下动作中记录日志：
  - 登录
  - 添加/编辑/删除服务器
  - SSH 连通性测试
  - 添加/编辑/删除 NAT 映射
  - 添加/编辑/删除外部节点
  - 发起安装/卸载节点
  - 创建/编辑/删除订阅
  - 重置订阅 token
- 实现当前用户操作日志接口：`GET /api/v1/operation-logs`
- 实现管理员查看所有操作日志接口：`GET /api/v1/admin/operation-logs`
- 操作日志不记录敏感明文。
- 增加基础测试覆盖用户隔离和管理员访问。

### 前端

- 安全中心页面从占位改为真实页面。
- 展示当前用户操作日志。
- 管理员后台增加操作日志汇总。

## 不包含范围

- 不实现复杂筛选和分页。
- 不记录敏感参数明文。
- 不实现日志导出。

## 验收标准

- 用户关键操作会写入操作日志。
- 普通用户只能查看自己的操作日志。
- 管理员可以查看所有操作日志。
- 前端安全中心能展示当前用户操作日志。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认
