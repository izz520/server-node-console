# 任务记录：登录注册错误提示优化

## 任务状态

已完成

## 任务背景

用户注册或登录失败时，后端已经返回具体错误，例如用户名/邮箱重复、账号或密码错误等。但前端当前多使用固定文案，用户排查问题时信息不足。

## 本次任务目标

让登录和注册页面优先展示后端返回的错误信息，并保留兜底提示。

## 本次调整范围

### 前端

- 登录失败展示后端错误信息。
- 注册失败展示后端错误信息。
- 增加本地 API 错误解析 helper，兼容 Axios 错误结构。

## 不包含范围

- 不调整后端错误码。
- 不做字段级表单校验提示。

## 验收标准

- 用户名/邮箱重复时，注册页能显示后端返回的错误。
- 登录账号或密码错误时，登录页能显示后端返回的错误。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 后端错误当前为英文短语，后续可统一做中文错误码映射。

## 完成记录

- 已新增通用 API 错误解析 helper。
- 登录页已优先展示后端返回的错误信息。
- 注册页已优先展示后端返回的错误信息。
- 服务器、节点、订阅页面已复用统一错误解析 helper。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
