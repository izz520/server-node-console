# 任务记录：异步任务状态自动刷新

## 任务状态

已完成

## 任务背景

`PRD.md` 要求安装/卸载任务异步执行，用户应能看到任务状态变化，并能查看任务日志。当前任务、节点和概览页面主要依赖进入页面时的列表请求，安装/卸载在后台执行后，前端状态可能不会及时刷新。

## 本次任务目标

为异步相关页面增加轻量轮询，让节点状态、任务状态和任务日志在安装/卸载过程中自动更新。

## 本次调整范围

### 前端

- 协议节点页面自动刷新节点列表。
- 任务日志页面自动刷新任务列表和选中任务详情。
- 概览页自动刷新最近节点和任务统计。
- 保持现有 API 不变。

## 不包含范围

- 不引入 WebSocket/SSE。
- 不新增后端推送接口。
- 不做复杂的“仅运行中任务才轮询”优化。

## 验收标准

- 安装/卸载任务执行期间，前端无需手动刷新即可看到状态变化。
- 任务详情日志会周期性更新。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 第一版使用 5 秒轮询，后续可根据任务状态优化为按需轮询或服务端推送。
