# 任务记录：识别 argosbx 输出中的安装失败

## 任务状态

已完成

## 任务背景

真实 AnyTLS 安装验证中，后端任务状态显示成功，但 argosbx 输出包含 `Argosbx脚本进程未启动，安装失败`。说明该脚本在部分失败场景下仍可能以退出码 0 结束，后端如果只依赖 SSH 命令退出码，会把实际失败的安装误判为成功。

## 本次任务目标

在安装任务中检查 argosbx 输出内容，识别明确失败标记；一旦发现失败标记，任务应进入失败态，节点状态应为安装失败，并把原因写入任务日志。

## 本次调整范围

### 后端

- 增加 argosbx 输出失败检测逻辑。
- 安装任务在 SSH 命令成功返回后继续检查输出。
- 增加输出检测单元测试。

### 验证

- 运行 `go test ./...`。
- 重新跑 AnyTLS 真实安装验证，确认脚本输出失败时后端不再误判为成功。

## 不包含范围

- 不修改 argosbx 脚本。
- 不保证当前 Alpine 服务器一定能成功安装 AnyTLS。
- 不解析所有可能的脚本文案，只覆盖本次真实验证暴露的明确失败标记，并保留可扩展结构。

## 验收标准

- 输出包含 `安装失败` 时，安装任务失败。
- 输出正常时，现有成功流程不受影响。
- `go test ./...` 通过。

## 执行结果

- 已新增 argosbx 安装输出失败标记检测。
- 安装任务在 SSH 命令退出码为 0 后，会继续检查输出中的 `安装失败` / `install failed`。
- 单元测试覆盖失败输出和正常输出。
- 后端测试通过。
- 真实服务器复测时远端命令再次阻塞，未能在该轮拿到脚本输出；本任务的输出误判问题已通过单元测试和前一轮真实输出复现完成修复。
