# 任务记录：前端管理员路由守卫

## 任务状态

已完成

## 任务背景

管理员接口已经由后端 `AdminOnly` 中间件保护，普通用户无法访问管理员数据。但前端 `/admin` 路由当前没有角色守卫，普通用户手动输入地址时仍会进入管理员页面并触发一组 403 请求，体验不够清晰。

## 本次任务目标

为前端管理员页面增加角色守卫，普通用户直接访问 `/admin` 时重定向回概览页。

## 本次调整范围

### 前端

- 增加 `RequireAdmin` 路由组件。
- `/admin` 路由包裹管理员守卫。
- 继续保留后端权限校验作为真正的安全边界。

## 不包含范围

- 不改变后端管理员权限逻辑。
- 不实现管理员登录入口区分。

## 验收标准

- 普通用户无法在前端进入管理员页面。
- 管理员用户可以正常访问管理员页面。
- `pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 前端守卫只改善体验，权限安全仍以后端为准。

## 完成记录

- 已新增 `RequireAdmin` 路由守卫。
- `/admin` 路由已限制为管理员角色访问。
- 普通用户手动访问 `/admin` 会重定向到概览页。

## 验证结果

- `pnpm check` 通过。
- `pnpm build` 通过。
