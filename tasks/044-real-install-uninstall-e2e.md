# 任务记录：真实服务器协议安装/卸载端到端验收

## 任务状态

已完成

## 任务背景

真实服务器 SSH 连通性已经通过。为了验证 PRD 第一期的核心闭环，需要通过后端 API 跑一轮真实端到端流程：添加服务器、创建安装任务、执行协议安装、查看任务日志、卸载清理。

## 本次任务目标

使用测试服务器执行一轮真实协议安装/卸载验收。根据用户最新指令，选择 `AnyTLS` 协议，安装成功后执行卸载清理。

## 本次调整范围

### 验证

- 使用后端 API 创建测试用户并登录。
- 使用后端 API 添加测试服务器。
- 使用后端 API 发起 `AnyTLS` 安装任务。
- 轮询任务状态和任务日志。
- 安装成功后发起卸载任务并轮询结果。

## 不包含范围

- 不把 SSH 密码写入任务文档或仓库文件。
- 不修改产品代码，除非验收暴露必须修复的问题。
- 不保留远端协议节点，安装成功后会尝试卸载清理。

## 验收标准

- 后端能通过 API 完成真实服务器 SSH 保存和连通性验证。
- 安装任务能真实执行并产生任务日志。
- 安装成功后可执行卸载任务。
- 如果安装或卸载失败，需要记录失败原因。

## 风险与待确认

- argosbx 脚本会真实修改远端服务器。
- 远端网络、系统环境、脚本源可用性可能影响结果。
- 如卸载失败，可能需要人工登录服务器清理。

## 执行结果

- 已通过后端 API 完成测试用户注册、服务器添加、SSH 连通性验证和 AnyTLS 安装任务创建。
- 初始安装失败暴露出远端缺少 `bash` 和命令 shell 兼容性问题，已在任务 `045` 修复。
- 随后暴露 SSH 命令执行阶段没有 timeout 的问题，已在任务 `046` 修复。
- 一轮 AnyTLS 安装返回 argosbx 输出 `Argosbx脚本进程未启动，安装失败`，但脚本退出码为 0，暴露后端误判成功问题，已在任务 `047` 修复。
- 安装成功误判后已执行卸载任务，卸载任务返回成功。
- 当前真实服务器为 Alpine Linux 3.21；后续复测中 argosbx 安装过程再次长时间阻塞。已停止本地临时后端，并将本地测试数据中的运行中任务标记为失败，避免页面残留安装中状态。

## 结论

本次真实 AnyTLS 验收完成了系统链路验证，并修复了 3 个真实环境问题；但该测试服务器上的 argosbx AnyTLS 未确认真实安装成功。下一步建议优先增加 SSH 流式日志/心跳日志，定位 argosbx 在 Alpine 环境下的阻塞点。
