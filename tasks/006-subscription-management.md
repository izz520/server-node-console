# 任务记录：订阅管理与订阅链接输出

## 任务状态

已完成

## 任务背景

根据 `PRD.md`，用户可以创建订阅，选择一个或多个节点，并生成带随机 token 的订阅链接。订阅是服务器、NAT 映射和协议节点管理之后的核心输出能力。当前外部节点基础管理已完成，可以开始实现订阅管理。

## 本次任务目标

实现普通用户订阅的创建、编辑、删除、启用/禁用、token 重置，以及公开订阅链接 `/sub/{token}` 的基础输出能力。

## 本次调整范围

### 后端

- 实现订阅列表接口：`GET /api/v1/subscriptions`
- 实现订阅详情接口：`GET /api/v1/subscriptions/:id`
- 实现订阅创建接口：`POST /api/v1/subscriptions`
- 实现订阅编辑接口：`PUT /api/v1/subscriptions/:id`
- 实现订阅删除接口：`DELETE /api/v1/subscriptions/:id`
- 实现订阅 token 重置接口：`POST /api/v1/subscriptions/:id/reset-token`
- 实现公开订阅访问接口：`GET /sub/:token`
- 创建/编辑订阅时校验节点属于当前用户。
- 订阅可包含多个节点。
- 删除订阅后 token 失效。
- token 重置后旧链接失效。
- 禁用订阅后公开链接不可正常使用。
- 支持基础格式输出：`sing-box`、`clash-mihomo`、`v2rayn`、`shadowrocket`、`base64`。

### 前端

- 实现订阅管理页面。
- 展示当前用户自己的订阅列表。
- 支持创建、编辑、删除订阅。
- 支持选择多个节点。
- 支持选择订阅格式。
- 支持启用/禁用订阅。
- 支持重置 token。
- 展示订阅链接。

## 不包含范围

- 不保证所有客户端格式的完整生产级配置。
- 不实现流量统计、过期时间、套餐配额。
- 不实现订阅访问日志。

## 验收标准

- 用户可以查看自己的订阅列表。
- 用户可以创建包含多个节点的订阅。
- 用户可以编辑订阅名称、格式、节点列表和启用状态。
- 用户可以删除订阅。
- 用户可以重置订阅 token。
- 旧 token 在重置后失效。
- 禁用订阅后公开链接不可正常使用。
- 用户不能选择或操作其他用户的节点和订阅。
- 前端订阅页面可以完成订阅 CRUD 和 token 重置。
- `go test ./...`、`pnpm check`、`pnpm build` 通过。

## 风险与待确认

- 第一版订阅内容输出以基础可读结构为主，后续可针对每个客户端做完整字段映射。
- `Subscription.TokenHash` 当前字段名为 hash，但第一期会保存 token 的 SHA-256 哈希，前端只在创建和重置时拿到明文 token。
