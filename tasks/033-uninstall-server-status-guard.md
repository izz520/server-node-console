# 任务记录：卸载协议前校验服务器状态

## 任务状态

已完成

## 任务背景

协议卸载同样需要通过 SSH 登录服务器执行脚本。`PRD.md` 中服务器“正常”状态表示可执行安装/卸载，连接失败或禁用状态不应继续创建卸载任务。当前卸载接口未校验服务器状态。

## 本次任务目标

在发起协议卸载前校验所属服务器状态，只有正常服务器允许创建卸载任务。

## 本次调整范围

### 后端

- `UninstallNode` 加载节点所属服务器。
- 服务器不存在或非正常状态时拒绝创建卸载任务。
- 增加测试覆盖非正常服务器不能卸载节点。

### 前端

- 复用现有错误提示和中文映射。

## 不包含范围

- 不自动触发 SSH 测试。
- 不改变卸载失败后的状态处理。

## 验收标准

- 正常服务器上的系统节点可以发起卸载。
- 连接失败服务器上的系统节点不能发起卸载。
- `go test ./...` 通过。

## 风险与待确认

- 如果服务器已永久不可达，用户需要后续“仅删除记录”类能力；第一期按 PRD 自动执行卸载脚本处理。
