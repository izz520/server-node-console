# 任务记录：删除节点时清理订阅关联

## 任务状态

已完成

## 任务背景

节点删除当前只软删除 `protocol_nodes` 记录，没有同步清理 `subscription_nodes` 关联表。这样订阅的 `nodeCount` 可能仍包含已删除节点，后续订阅编辑也会保留无效关联。

## 本次任务目标

删除节点时同步移除该节点与订阅的关联，保证订阅列表和订阅内容不会残留已删除节点。

## 本次调整范围

### 后端

- `DeleteNode` 改为事务。
- 删除节点前先删除对应 `subscription_nodes` 关联。
- 增加测试覆盖节点删除后订阅关联清理。

## 不包含范围

- 不删除订阅本身。
- 不自动重置订阅 token。
- 不改变已卸载节点过滤规则。

## 验收标准

- 删除节点后，订阅关联表不再保留该节点。
- 订阅 nodeCount 不再统计已删除节点。
- `go test ./...` 通过。

## 风险与待确认

- 删除节点会影响订阅内容，前端已增加删除确认提示。

## 完成记录

- `DeleteNode` 已改为事务处理。
- 删除节点前会先删除 `subscription_nodes` 中对应节点关联。
- 已增加测试覆盖删除节点后订阅关联和 nodeCount 清理。

## 验证结果

- `go test ./...` 通过。
