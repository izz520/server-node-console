# 服务器管理以及 sing-box 节点安装和订阅管理系统 PRD

## 1. 产品定位

本系统是一个面向多用户的 SaaS 平台，用于管理用户自己的服务器、通过 SSH 自动安装/卸载 sing-box 相关协议节点，并基于已安装节点或外部导入节点生成多客户端可用的订阅链接。

### 1.1 目标用户

- 普通用户：注册并登录系统后，管理自己账号下的服务器、协议节点和订阅。
- 管理员：登录后台后，只读查看所有用户的服务器、协议节点和订阅数据，用于平台运维和问题排查。

### 1.2 产品目标

- 降低用户部署 sing-box 节点的操作门槛。
- 支持用户集中管理多台服务器、多种协议节点和多个订阅。
- 支持 NAT 服务器端口映射记录，便于生成正确的订阅链接。
- 支持多种客户端订阅格式，覆盖常见代理客户端。

## 3. 角色与权限

### 3.1 普通用户

普通用户只能访问和管理自己账号下的数据。

权限范围：

- 注册账号、登录系统、退出登录。
- 添加、编辑、删除自己的服务器。
- 添加、编辑、删除自己服务器的 NAT 端口映射记录。
- 在自己的服务器上安装、卸载协议节点。
- 手动添加自己的外部已安装节点。
- 通过分享链接导入自己的外部节点。
- 创建、编辑、删除、启用、禁用自己的订阅。
- 重置自己的订阅 token。
- 查看自己的安装/卸载任务状态和任务日志。

### 3.2 管理员

管理员拥有平台级只读权限。

权限范围：

- 查看所有用户列表。
- 查看所有用户的服务器、协议节点和订阅。
- 查看所有用户的安装/卸载任务状态和任务日志。

权限限制：

- 管理员第一期不允许编辑或删除用户数据。
- 管理员第一期不允许代替用户安装、卸载协议节点。
- 管理员第一期不允许查看 SSH 密码、SSH 私钥、节点密钥等敏感明文。

## 4. 用户管理

### 4.1 注册

用户可通过用户名/邮箱和密码注册账号。

规则：

- 第一期开启自由注册。
- 不需要邮箱验证码。
- 用户名或邮箱不可重复。
- 密码需要进行安全哈希后保存。

### 4.2 登录

用户可通过用户名/邮箱和密码登录系统。

规则：

- 登录成功后返回 JWT。
- 前端使用 JWT 访问需要鉴权的接口。
- JWT 过期后用户需要重新登录，刷新机制可后续迭代。

### 4.3 数据隔离

所有普通用户数据必须按用户维度隔离。

隔离对象包括：

- 服务器
- NAT 端口映射
- 协议节点
- 外部节点
- 订阅
- 安装/卸载任务
- 任务日志

## 5. 服务器管理

### 5.1 添加服务器

用户可以添加自己的服务器，用于后续安装协议节点。

服务器字段建议：

- 服务器名称
- 主机地址/IP
- SSH 端口
- SSH 用户名
- SSH 认证方式：密码或私钥
- SSH 密码，认证方式为密码时填写
- SSH 私钥，认证方式为私钥时填写
- 服务器地区，可选
- 标签，可选
- 备注，可选
- 状态：正常、连接失败、禁用

### 5.2 SSH 凭据保存规则

- SSH 密码和 SSH 私钥需要保存到数据库，方便后续自动安装和卸载协议。
- SSH 密码和 SSH 私钥必须加密保存。
- 页面不允许再次查看 SSH 密码或 SSH 私钥明文。
- 页面只显示脱敏信息，例如“已保存密码”或“已保存私钥”。
- 用户可以重新填写 SSH 密码或 SSH 私钥来覆盖旧凭据。

### 5.3 SSH 连通性测试

添加服务器时，系统需要在保存前自动测试 SSH 连通性。

测试内容：

- 主机地址/IP 是否可访问。
- SSH 端口是否可连接。
- SSH 用户名和密码/私钥是否可登录。

用户也可以在服务器详情或服务器列表中手动点击“测试连通性”。

测试结果：

- 测试成功：服务器状态更新为正常。
- 测试失败：提示失败原因，服务器不可保存或保存为连接失败状态，具体交互由实现时确认。

### 5.4 编辑服务器

用户可以编辑自己账号下的服务器信息。

规则：

- 修改 SSH 地址、端口、用户名或凭据后，应支持重新测试连通性。
- 修改 SSH 凭据时，新凭据覆盖旧凭据。
- 已关联节点和订阅的服务器仍可编辑基础信息。

### 5.5 删除服务器

用户可以删除自己账号下的服务器。

删除规则待实现时确认，建议第一期采用保护策略：

- 如果服务器下存在已安装节点或订阅引用，不允许直接删除。
- 用户需要先卸载/删除节点，并从订阅中移除相关节点后，再删除服务器。

## 6. NAT 端口映射管理

NAT 端口映射是系统内的记录功能，不需要真实修改服务器网络配置。

### 6.1 使用场景

例如用户的 NAT 服务器上 AnyTLS 实际监听端口为 8000，但公网可访问端口为 9000。系统记录该映射关系后，生成订阅链接时应使用公网对外端口 9000。

### 6.2 映射字段建议

- 所属服务器
- 映射名称
- 协议类型，可选，例如 TCP/UDP
- 服务实际监听端口
- 对外访问端口
- 备注

### 6.3 订阅生成规则

- 每个节点可以配置服务实际监听端口和对外访问端口。
- 生成订阅时，如果节点存在对外访问端口，则优先使用对外访问端口。
- 如果节点不存在对外访问端口，则使用服务实际监听端口。

## 7. sing-box 节点管理

### 7.1 支持协议范围

第一期根据 yonggekkk/argosbx 支持的协议范围实现：

- AnyTLS
- Any-reality
- Vless-xhttp-reality-vision-enc
- Vless-tcp-reality-vision
- Vless-xhttp-vision-enc
- Vless-ws-vision-enc
- Shadowsocks-2022
- Hysteria2
- Tuic
- Socks5
- Vmess-ws
- Argo 临时隧道
- Argo 固定隧道

参考资料：

- https://github.com/yonggekkk/argosbx
- https://yonggekkk.github.io/argosbx

### 7.2 安装协议节点

用户可以选择协议、选择自己的服务器，并配置协议参数后发起安装。

安装方式：

- 系统通过已保存的 SSH 凭据自动登录服务器。
- 系统根据用户选择的协议和参数生成安装脚本。
- 后端自动执行安装脚本。
- 安装过程以异步任务形式运行。

协议参数规则：

- 能自动生成的参数尽量自动生成，例如 UUID、密码、密钥等。
- 前端可提供输入框，让用户手动覆盖自动生成的默认值。
- 用户不填写时，系统使用自动生成值或协议默认值。
- 不同协议需要维护独立的参数模板、默认值和校验规则。

### 7.3 安装任务

协议安装耗时较长，必须通过任务机制处理。

用户点击安装后：

- 后端创建安装任务。
- 前端立即在协议列表中显示该协议记录。
- 初始状态显示为安装中。
- 用户可以进入任务详情查看安装日志。

任务状态：

- 排队中
- 安装中
- 安装成功
- 安装失败

任务日志：

- 记录脚本生成时间。
- 记录 SSH 连接结果。
- 记录脚本执行输出。
- 记录错误信息。
- 支持用户查看完整日志。

### 7.4 安装成功后的节点信息

安装成功后，系统保存协议节点信息。

节点字段建议：

- 节点名称
- 所属用户
- 所属服务器
- 协议类型
- 服务监听端口
- 对外访问端口，可选
- 协议参数
- 订阅生成所需参数
- 安装方式：系统安装、外部导入
- 状态：安装中、安装成功、安装失败、卸载中、已卸载
- 创建时间
- 更新时间

### 7.5 卸载协议节点

用户可以卸载自己服务器上已经安装的协议节点。

卸载方式：

- 系统通过 SSH 自动登录服务器。
- 系统根据协议和服务器信息生成卸载脚本。
- 后端自动执行卸载脚本。
- 卸载过程以异步任务形式运行。
- 卸载成功后将节点标记为已卸载，并从后续订阅生成中排除。

卸载任务状态：

- 排队中
- 卸载中
- 卸载成功
- 卸载失败

### 7.6 外部节点添加

用户可以添加外部已安装节点，用于统一生成订阅。

支持方式：

- 手动填写节点参数。
- 粘贴分享链接导入。

外部节点规则：

- 外部节点可以不绑定系统内服务器。
- 外部节点需要保存生成订阅所需的完整参数。
- 外部节点状态默认为可用。
- 外部节点可以加入任意该用户自己的订阅。

## 8. 订阅管理

### 8.1 创建订阅

用户可以创建自己的订阅，并选择一个或多个节点加入订阅。

规则：

- 一个订阅可以包含多个节点，不限制数量。
- 一个节点可以被多个订阅引用。
- 订阅可以包含系统安装节点和外部导入节点。

订阅字段建议：

- 订阅名称
- 所属用户
- 订阅 token
- 启用状态
- 订阅格式
- 包含节点列表
- 备注
- 创建时间
- 更新时间

### 8.2 订阅格式

第一期支持以下客户端格式：

- sing-box
- Clash / Mihomo
- v2rayN
- Shadowrocket
- 通用 Base64 订阅

### 8.3 订阅链接

每个订阅生成一个随机 token。

链接形式：

- `/sub/{token}`

规则：

- token 必须足够随机，避免被猜测。
- 用户可以重置订阅 token。
- 重置 token 后，旧订阅链接立即失效。
- 访问启用状态的订阅链接时，返回对应格式的订阅内容。
- 访问禁用状态的订阅链接时，返回不可用提示或空订阅，具体响应格式实现时确认。

### 8.4 编辑订阅

用户可以编辑自己的订阅。

可编辑内容：

- 订阅名称
- 订阅格式
- 包含节点
- 启用/禁用状态
- 备注

### 8.5 删除订阅

用户可以删除自己的订阅。

规则：

- 删除订阅后，对应订阅 token 立即失效。
- 删除订阅不影响节点本身。

## 10. 安全要求

### 10.1 敏感信息保护

以下信息必须加密存储或安全处理：

- SSH 密码
- SSH 私钥
- 协议节点密钥
- UUID、密码、公私钥等协议敏感参数
- 订阅 token

页面展示规则：

- SSH 密码和 SSH 私钥不允许明文回显。
- 敏感协议参数默认脱敏展示。
- 用户可以重新生成或重新填写敏感参数。

### 10.2 访问控制

- 普通用户只能访问自己的数据。
- 管理员只能只读查看所有用户数据。
- 所有写操作必须校验数据归属。
- 订阅链接通过 token 访问，不需要登录，但 token 必须不可预测。

### 10.3 操作日志

建议记录关键操作日志：

- 用户登录
- 添加/编辑/删除服务器
- 测试 SSH 连通性
- 安装/卸载协议节点
- 添加外部节点
- 创建/编辑/删除订阅
- 重置订阅 token

## 11. 核心业务流程

### 11.1 添加服务器流程

1. 用户填写服务器信息和 SSH 凭据。
2. 系统自动测试 SSH 连通性。
3. 测试成功后保存服务器和加密后的 SSH 凭据。
4. 服务器进入正常状态。

### 11.2 安装协议流程

1. 用户选择服务器。
2. 用户选择协议。
3. 系统生成默认协议参数。
4. 用户可手动修改参数。
5. 用户点击安装。
6. 后端创建安装任务。
7. 协议列表显示安装中。
8. 后端通过 SSH 执行安装脚本。
9. 安装成功后节点状态更新为安装成功。
10. 安装失败后节点状态更新为安装失败，并展示任务日志。

### 11.3 卸载协议流程

1. 用户选择需要卸载的节点。
2. 用户确认卸载。
3. 后端创建卸载任务。
4. 节点状态更新为卸载中。
5. 后端通过 SSH 执行卸载脚本。
6. 卸载成功后节点状态更新为已卸载，并从后续订阅生成中排除。
7. 卸载失败后保留节点记录，并展示失败日志。

### 11.4 添加外部节点流程

1. 用户选择手动填写或粘贴分享链接导入。
2. 系统解析或校验节点参数。
3. 用户确认保存。
4. 外部节点进入可用状态。
5. 用户可以将该节点加入订阅。

### 11.5 创建订阅流程

1. 用户填写订阅名称。
2. 用户选择订阅格式。
3. 用户选择一个或多个节点。
4. 系统生成订阅 token。
5. 系统生成订阅链接。
6. 用户复制订阅链接到客户端使用。

## 12. 验收标准

### 12.1 用户和权限

- 用户可以通过用户名/邮箱和密码注册。
- 用户可以登录系统并获取访问权限。
- 普通用户无法查看或操作其他用户的数据。
- 管理员可以只读查看所有用户的数据。
- 管理员无法编辑或删除用户数据。

### 12.2 服务器管理

- 用户可以添加使用密码认证的服务器。
- 用户可以添加使用私钥认证的服务器。
- 添加服务器时系统会自动测试 SSH 连通性。
- 用户可以手动触发 SSH 连通性测试。
- SSH 密码和私钥不会在页面明文展示。
- 用户可以重新填写 SSH 凭据并覆盖旧凭据。

### 12.3 NAT 端口映射

- 用户可以为节点配置服务监听端口和对外访问端口。
- 生成订阅时，如果存在对外访问端口，订阅内容使用对外访问端口。
- 生成订阅时，如果不存在对外访问端口，订阅内容使用服务监听端口。

### 12.4 协议安装和卸载

- 用户可以选择自己的服务器安装协议节点。
- 安装任务创建后，协议列表立即显示该节点为安装中。
- 安装成功后，节点状态更新为安装成功。
- 安装失败后，节点状态更新为安装失败，并可查看失败日志。
- 用户可以卸载已安装节点。
- 卸载时系统会通过 SSH 执行卸载脚本。
- 卸载成功后，节点状态更新为已卸载，并从后续订阅生成中排除。

### 12.5 外部节点

- 用户可以手动填写外部节点参数并保存。
- 用户可以粘贴分享链接导入外部节点。
- 外部节点可以加入订阅。

### 12.6 订阅管理

- 用户可以创建包含多个节点的订阅。
- 用户可以编辑订阅名称、格式、节点列表和启用状态。
- 用户可以删除订阅。
- 每个订阅拥有独立随机 token。
- 用户可以重置订阅 token。
- token 重置后旧链接失效。
- 禁用订阅后，订阅链接不可正常使用。
- 订阅内容支持 sing-box、Clash/Mihomo、v2rayN、Shadowrocket 和通用 Base64 格式。

## 13. 第一期待确认事项

以下内容不阻塞第一版需求落地，但建议在进入详细设计前确认：

- 服务器删除时，关联节点和订阅的处理策略是否采用“有引用则禁止删除”。
- 安装失败的节点是否允许用户编辑参数后重试。
- 订阅禁用后的响应格式，是返回 403、空内容，还是错误说明。
- 是否需要用户禁用功能。
- 是否需要登录失败次数限制和账号锁定策略。
- 是否需要套餐、配额或计费能力，当前第一期不做限制。
- 是否需要支持刷新 token，当前第一期可只使用登录 JWT。
