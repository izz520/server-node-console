# 服务器管理与 sing-box 节点订阅管理系统 PRD

版本：V1.0  
日期：2026-05-23  
产品形态：多用户 SaaS 平台

## 1. 文档目的

本文档用于明确“服务器管理与 sing-box 节点订阅管理系统”的产品范围、用户角色、核心功能、业务规则、安全要求和验收标准，作为后续 UI 设计、数据库设计、接口设计、研发实现和测试验收的依据。

## 2. 产品概述

本系统面向多用户提供服务器、协议节点和订阅的集中管理能力。用户注册并登录后，可以添加自己的服务器，系统通过 SSH 自动连接服务器并执行 sing-box 相关协议的安装/卸载脚本。用户也可以手动添加外部已安装节点，最终将系统安装节点和外部节点组合成不同客户端可用的订阅链接。

系统同时支持 NAT 服务器端口映射记录。对于 NAT 场景，节点实际监听端口和公网对外端口可能不同，系统在生成订阅时需要优先使用对外访问端口。

## 3. 产品目标

- 让用户无需手动登录服务器，即可完成 sing-box 协议节点的安装和卸载。
- 让用户集中管理多台服务器、多种协议节点和多个订阅。
- 支持外部已安装节点导入，避免用户重复部署。
- 支持 NAT 端口映射记录，确保订阅链接中的地址和端口可真实访问。
- 支持主流客户端订阅格式，降低用户配置客户端的成本。
- 保证 SaaS 多用户数据隔离和敏感凭据安全。

## 4. 用户角色

### 4.1 普通用户

普通用户是系统的主要使用者。用户可以注册账号并登录系统，管理自己账号下的服务器、协议节点、外部节点、订阅和任务日志。

普通用户权限：

- 注册、登录、退出登录。
- 添加、编辑、删除自己的服务器。
- 添加、编辑、删除自己服务器下的 NAT 端口映射记录。
- 在自己的服务器上安装和卸载协议节点。
- 查看自己的安装/卸载任务状态和任务日志。
- 手动添加外部节点。
- 通过分享链接导入外部节点。
- 创建、编辑、删除、启用、禁用自己的订阅。
- 重置自己的订阅 token。

### 4.2 管理员

管理员用于平台运维和问题排查。管理员第一期只具备只读权限。

管理员权限：

- 查看所有用户列表。
- 查看所有用户的服务器。
- 查看所有用户的协议节点和外部节点。
- 查看所有用户的订阅。
- 查看所有用户的安装/卸载任务和任务日志。

管理员限制：

- 不允许编辑或删除用户数据。
- 不允许代替用户安装或卸载协议节点。
- 不允许查看 SSH 密码、SSH 私钥、节点密钥等敏感明文。

## 7. 功能需求

### 7.1 用户管理

#### 7.1.1 注册

用户可以通过用户名/邮箱和密码注册账号。

业务规则：

- 第一期开启自由注册。
- 不需要邮箱验证码。
- 用户名不可重复。
- 邮箱不可重复。
- 密码必须安全哈希后存储。

验收标准：

- 用户可以使用用户名、邮箱和密码完成注册。
- 重复用户名或重复邮箱注册失败。
- 数据库不保存明文密码。

#### 7.1.2 登录

用户可以使用用户名/邮箱和密码登录系统。

业务规则：

- 登录成功后返回 JWT。
- 登录失败时返回明确错误提示。
- 前端携带 JWT 访问需要鉴权的接口。
- JWT 过期后用户需要重新登录。

验收标准：

- 用户可以使用正确账号密码登录。
- 用户使用错误账号密码无法登录。
- 未登录用户无法访问需要鉴权的接口。

#### 7.1.3 多用户数据隔离

普通用户只能访问自己账号下的数据。

隔离对象：

- 服务器
- NAT 端口映射
- 协议节点
- 外部节点
- 订阅
- 安装/卸载任务
- 任务日志

验收标准：

- 普通用户无法查看其他用户的数据。
- 普通用户无法通过接口操作其他用户的数据。
- 所有写操作均校验数据归属。

### 7.2 服务器管理

#### 7.2.1 添加服务器

用户可以添加自己的服务器，用于后续自动安装和卸载协议节点。

字段要求：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 服务器名称 | 是 | 用户自定义名称 |
| 主机地址/IP | 是 | SSH 连接地址 |
| SSH 端口 | 是 | 默认可为 22 |
| SSH 用户名 | 是 | 登录服务器使用 |
| SSH 认证方式 | 是 | 密码或私钥 |
| SSH 密码 | 条件必填 | 认证方式为密码时必填 |
| SSH 私钥 | 条件必填 | 认证方式为私钥时必填 |
| 服务器地区 | 否 | 用户自定义 |
| 标签 | 否 | 用户自定义 |
| 备注 | 否 | 用户自定义 |
| 状态 | 系统生成 | 正常、连接失败、禁用 |

业务规则：

- 保存前必须自动测试 SSH 连通性。
- SSH 连接成功后才允许保存为正常状态。
- SSH 连接失败时需要展示失败原因。
- 失败时是否允许保存为连接失败状态，进入详细设计前确认。

验收标准：

- 用户可以添加密码认证服务器。
- 用户可以添加私钥认证服务器。
- 添加服务器时系统会自动测试 SSH 连通性。
- 测试失败时用户可以看到失败原因。

#### 7.2.2 SSH 凭据安全

业务规则：

- SSH 密码和 SSH 私钥需要保存到数据库，方便后续自动安装和卸载。
- SSH 密码和 SSH 私钥必须加密保存。
- 页面不允许回显 SSH 密码或 SSH 私钥明文。
- 页面只展示脱敏状态，例如“已保存密码”或“已保存私钥”。
- 用户可以重新填写 SSH 密码或 SSH 私钥来覆盖旧凭据。

验收标准：

- 数据库不保存明文 SSH 密码。
- 数据库不保存明文 SSH 私钥。
- 页面不会展示 SSH 凭据明文。
- 用户可以更新 SSH 凭据。

#### 7.2.3 测试连通性

用户可以在服务器详情或服务器列表中手动触发 SSH 连通性测试。

测试内容：

- 主机地址/IP 是否可访问。
- SSH 端口是否可连接。
- SSH 用户名和密码/私钥是否可登录。

验收标准：

- 用户可以手动触发连通性测试。
- 测试成功后服务器状态更新为正常。
- 测试失败后服务器状态更新为连接失败，并显示失败原因。

#### 7.2.4 编辑服务器

业务规则：

- 用户只能编辑自己的服务器。
- 修改 SSH 地址、端口、用户名或凭据后，应支持重新测试连通性。
- 修改 SSH 凭据时，新凭据覆盖旧凭据。
- 已关联节点和订阅的服务器仍可编辑基础信息。

验收标准：

- 用户可以编辑自己的服务器信息。
- 用户不能编辑其他用户的服务器。
- 修改凭据后，后续安装/卸载使用新凭据。

#### 7.2.5 删除服务器

建议第一期采用保护策略。

业务规则：

- 如果服务器下存在已安装节点或订阅引用，不允许直接删除。
- 用户需要先卸载/删除节点，并从订阅中移除相关节点后，再删除服务器。

验收标准：

- 无关联数据的服务器可以删除。
- 存在节点或订阅引用的服务器删除失败，并提示用户先处理关联数据。

### 7.3 NAT 端口映射管理

NAT 端口映射只作为系统内记录，不真实修改服务器网络配置。

使用场景：

用户的 NAT 服务器上某协议实际监听端口为 8000，但公网可访问端口为 9000。系统记录映射关系后，生成订阅链接时使用 9000。

字段要求：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 所属服务器 | 是 | 用户自己的服务器 |
| 映射名称 | 是 | 用户自定义名称 |
| 协议类型 | 否 | TCP/UDP |
| 服务实际监听端口 | 是 | 节点在服务器上监听的端口 |
| 对外访问端口 | 是 | 公网访问端口 |
| 备注 | 否 | 用户自定义 |

订阅生成规则：

- 每个节点可以配置服务实际监听端口和对外访问端口。
- 如果节点存在对外访问端口，订阅内容优先使用对外访问端口。
- 如果节点不存在对外访问端口，订阅内容使用服务实际监听端口。

验收标准：

- 用户可以维护服务器的 NAT 端口映射记录。
- 订阅生成时优先使用对外访问端口。
- NAT 映射不会触发真实服务器网络配置变更。

#### 7.4.2 协议安装

用户可以选择自己的服务器、选择协议、确认或修改协议参数，然后发起安装。

业务规则：

- 系统通过已保存的 SSH 凭据自动登录服务器。
- 系统根据协议和参数生成安装脚本。
- 后端自动执行安装脚本。
- 安装过程以异步任务运行。
- 用户点击安装后，前端协议列表立即出现该节点，状态为安装中。
- 安装成功后节点状态更新为安装成功。
- 安装失败后节点状态更新为安装失败，并展示任务日志。

协议参数规则：

- 能自动生成的参数尽量自动生成，例如 UUID、密码、密钥等。
- 用户可以手动覆盖自动生成的默认值。
- 用户不填写时，系统使用自动生成值或协议默认值。
- 每个协议需要独立维护参数模板、默认值和校验规则。

验收标准：

- 用户可以选择自己的服务器安装协议。
- 安装任务创建后，协议列表显示安装中。
- 安装成功后，节点状态为安装成功。
- 安装失败后，节点状态为安装失败，并可查看失败日志。

#### 7.4.3 安装/卸载任务

协议安装和卸载需要通过后端任务系统执行。

任务状态：

- 排队中
- 执行中
- 成功
- 失败

安装任务节点状态：

- 安装中
- 安装成功
- 安装失败

卸载任务节点状态：

- 卸载中
- 已卸载

任务日志要求：

- 记录任务创建时间。
- 记录脚本生成时间。
- 记录 SSH 连接结果。
- 记录脚本执行输出。
- 记录错误信息。
- 支持用户查看完整日志。
- 管理员可只读查看所有用户任务日志。

验收标准：

- 安装/卸载任务可以异步执行。
- 用户可以看到任务状态变化。
- 用户可以查看自己的任务日志。
- 管理员可以只读查看所有任务日志。

#### 7.4.5 协议卸载

用户可以卸载自己服务器上已经安装的协议节点。

业务规则：

- 系统通过 SSH 自动登录服务器。
- 系统根据协议和服务器信息生成卸载脚本。
- 后端自动执行卸载脚本。
- 卸载过程以异步任务运行。
- 卸载成功后节点状态更新为已卸载。
- 已卸载节点从后续订阅生成中排除。
- 卸载失败后保留节点原状态或更新为卸载失败状态，需展示失败日志。

验收标准：

- 用户可以卸载已安装节点。
- 卸载时系统会通过 SSH 执行卸载脚本。
- 卸载成功后节点不再出现在有效订阅内容中。
- 卸载失败后用户可以查看失败原因。

### 7.5 外部节点管理

用户可以将外部已安装节点添加到系统，用于统一生成订阅。

支持方式：

- 手动填写节点参数。
- 粘贴分享链接导入。

业务规则：

- 外部节点可以不绑定系统内服务器。
- 外部节点需要保存生成订阅所需的完整参数。
- 外部节点状态默认为可用。
- 外部节点可以加入该用户自己的任意订阅。
- 用户只能管理自己的外部节点。

验收标准：

- 用户可以手动添加外部节点。
- 用户可以粘贴分享链接导入外部节点。
- 外部节点可以被加入订阅。
- 外部节点不会触发服务器安装任务。

### 7.6 订阅管理

#### 7.6.1 创建订阅

用户可以创建自己的订阅，并选择一个或多个节点加入订阅。

业务规则：

- 一个订阅可以包含多个节点，不限制数量。
- 一个节点可以被多个订阅引用。
- 订阅可以包含系统安装节点和外部导入节点。
- 已卸载节点、安装失败节点不应出现在有效订阅内容中。

字段要求：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 订阅名称 | 是 | 用户自定义 |
| 所属用户 | 系统生成 | 数据归属 |
| 订阅 token | 系统生成 | 随机且不可预测 |
| 启用状态 | 是 | 启用或禁用 |
| 订阅格式 | 是 | sing-box、Clash/Mihomo 等 |
| 包含节点列表 | 是 | 一个或多个节点 |
| 备注 | 否 | 用户自定义 |
| 创建时间 | 系统生成 |  |
| 更新时间 | 系统生成 |  |

验收标准：

- 用户可以创建包含多个节点的订阅。
- 用户可以选择系统安装节点和外部节点。
- 订阅只包含当前用户自己的节点。

#### 7.6.2 订阅格式

第一期支持以下客户端格式：

- sing-box
- Clash / Mihomo
- v2rayN
- Shadowrocket
- 通用 Base64 订阅

验收标准：

- 同一批节点可以按不同格式输出订阅内容。
- 客户端格式参数符合对应客户端要求。

#### 7.6.3 订阅链接和 token

每个订阅生成一个随机 token。

链接形式：

```text
/sub/{token}
```

业务规则：

- 订阅链接通过 token 访问，不要求用户登录。
- token 必须足够随机，避免被猜测。
- 用户可以重置订阅 token。
- 重置 token 后，旧链接立即失效。
- 删除订阅后，对应 token 立即失效。
- 启用状态下访问订阅链接，返回对应格式的订阅内容。
- 禁用状态下访问订阅链接，返回不可用提示或空订阅，响应格式进入详细设计前确认。

验收标准：

- 每个订阅拥有独立 token。
- 用户可以重置 token。
- 旧 token 在重置后无法访问。
- 删除订阅后原链接失效。
- 禁用订阅后订阅链接不可正常使用。

#### 7.6.4 编辑和删除订阅

可编辑内容：

- 订阅名称
- 订阅格式
- 包含节点
- 启用/禁用状态
- 备注

业务规则：

- 用户只能编辑自己的订阅。
- 删除订阅不影响节点本身。
- 删除订阅后 token 立即失效。

验收标准：

- 用户可以编辑自己的订阅。
- 用户可以删除自己的订阅。
- 用户不能编辑或删除其他用户的订阅。

## 8. 状态定义

### 8.1 服务器状态

| 状态 | 含义 |
| --- | --- |
| 正常 | SSH 可连接，可执行安装/卸载 |
| 连接失败 | 最近一次 SSH 测试失败 |
| 禁用 | 暂不允许用于安装/卸载，后续迭代 |

### 8.2 节点状态

| 状态 | 含义 |
| --- | --- |
| 安装中 | 安装任务正在执行 |
| 安装成功 | 节点安装成功，可用于订阅 |
| 安装失败 | 节点安装失败，不可用于订阅 |
| 卸载中 | 卸载任务正在执行 |
| 已卸载 | 节点已卸载，不再用于订阅 |
| 外部导入 | 外部节点，未通过系统安装 |

### 8.3 任务状态

| 状态 | 含义 |
| --- | --- |
| 排队中 | 任务已创建，等待执行 |
| 执行中 | 任务正在执行 |
| 成功 | 任务执行成功 |
| 失败 | 任务执行失败 |

### 8.4 订阅状态

| 状态 | 含义 |
| --- | --- |
| 启用 | 订阅链接可正常访问 |
| 禁用 | 订阅链接不可正常使用 |

## 9. 安全要求

### 9.1 敏感信息

以下信息必须加密存储或安全处理：

- SSH 密码
- SSH 私钥
- 协议节点密钥
- UUID、密码、公私钥等协议敏感参数
- 订阅 token

页面展示规则：

- SSH 密码和 SSH 私钥不允许明文回显。
- 协议敏感参数默认脱敏展示。
- 用户可以重新生成或重新填写敏感参数。

### 9.2 权限控制

- 普通用户只能访问自己的数据。
- 管理员只能只读查看所有用户数据。
- 所有写操作必须校验数据归属。
- 订阅链接可以匿名访问，但必须依赖不可预测 token。

### 9.3 操作日志

建议记录以下关键操作：

- 用户登录。
- 添加、编辑、删除服务器。
- 测试 SSH 连通性。
- 安装、卸载协议节点。
- 添加外部节点。
- 创建、编辑、删除订阅。
- 重置订阅 token。

## 10. 核心业务流程

### 10.1 添加服务器流程

1. 用户填写服务器信息和 SSH 凭据。
2. 系统自动测试 SSH 连通性。
3. 测试成功后保存服务器信息。
4. 系统加密保存 SSH 凭据。
5. 服务器状态更新为正常。

### 10.2 安装协议流程

1. 用户选择服务器。
2. 用户选择协议。
3. 系统生成默认协议参数。
4. 用户可手动修改参数。
5. 用户点击安装。
6. 后端创建安装任务。
7. 前端协议列表显示节点为安装中。
8. 后端通过 SSH 执行安装脚本。
9. 安装成功后节点状态更新为安装成功。
10. 安装失败后节点状态更新为安装失败，并展示任务日志。

### 10.3 卸载协议流程

1. 用户选择需要卸载的节点。
2. 用户确认卸载。
3. 后端创建卸载任务。
4. 节点状态更新为卸载中。
5. 后端通过 SSH 执行卸载脚本。
6. 卸载成功后节点状态更新为已卸载。
7. 已卸载节点从后续订阅生成中排除。
8. 卸载失败后展示失败日志。

### 10.4 添加外部节点流程

1. 用户选择手动填写或粘贴分享链接导入。
2. 系统解析或校验节点参数。
3. 用户确认保存。
4. 外部节点进入可用状态。
5. 用户可以将外部节点加入订阅。

### 10.5 创建订阅流程

1. 用户填写订阅名称。
2. 用户选择订阅格式。
3. 用户选择一个或多个节点。
4. 系统生成订阅 token。
5. 系统生成订阅链接。
6. 用户复制订阅链接到客户端使用。

## 11. 非功能需求

### 11.1 可用性

- 安装和卸载任务不能阻塞前端页面。
- 长耗时任务必须异步执行。
- 用户应能随时查看任务当前状态和日志。

### 11.2 可维护性

- 不同协议的参数模板、默认值、校验规则和脚本生成逻辑应相互隔离。
- 订阅格式生成逻辑应模块化，便于后续新增客户端格式。
- 任务系统应保留日志，便于排查安装失败原因。

### 11.3 安全性

- 密码、私钥、节点密钥等敏感信息必须加密存储。
- 所有普通用户接口必须校验用户数据归属。
- 管理员接口必须遵守只读限制。
- 订阅 token 必须不可预测，并支持重置失效。

## 12. 总体验收标准

- 用户可以注册、登录并进入系统。
- 普通用户只能管理自己的服务器、节点和订阅。
- 管理员可以只读查看所有用户数据。
- 用户可以添加服务器，并在保存前完成 SSH 连通性测试。
- SSH 密码和私钥加密保存，页面不显示明文。
- 用户可以在自己的服务器上发起协议安装任务。
- 协议安装任务会异步执行，并显示任务状态和日志。
- 安装成功的节点可以加入订阅。
- 安装失败的节点不会进入有效订阅内容。
- 用户可以卸载已安装节点。
- 已卸载节点不会进入后续订阅内容。
- 用户可以手动添加外部节点。
- 用户可以通过分享链接导入外部节点。
- 用户可以创建包含多个节点的订阅。
- 订阅支持 sing-box、Clash/Mihomo、v2rayN、Shadowrocket 和通用 Base64 格式。
- 每个订阅都有独立随机 token。
- 用户可以重置订阅 token，旧链接立即失效。
- 用户可以启用、禁用和删除订阅。
- NAT 场景下，订阅优先使用对外访问端口。

## 13. 待确认事项

以下事项不阻塞第一期 PRD，但建议进入详细设计或研发前确认：

- SSH 连通性测试失败时，是否允许保存为连接失败状态。
- 服务器删除时，是否严格采用“有节点或订阅引用则禁止删除”的保护策略。
- 安装失败的节点是否允许用户编辑参数后重试。
- 卸载失败时节点状态是否单独增加“卸载失败”。
- 订阅禁用后的响应格式，是返回 403、空内容，还是错误说明。
- 是否需要用户禁用功能。
- 是否需要登录失败次数限制和账号锁定策略。
- 是否需要刷新 token，当前第一期可只使用登录 JWT。
